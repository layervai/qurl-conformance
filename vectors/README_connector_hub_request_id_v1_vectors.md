# Connector Hub request-ID v1 vectors

`connector_hub_request_id_v1_vectors.json` freezes the private replay-key
derivation shared by Connector Hub workers. It is not a public SDK response or
a generic idempotency API.

Native SDK assignment remains NHP over UDP. Each initial, refresh, or explicit
credential-recovery assignment LST carries `usrData.request_nonce`: canonical
unpadded base64url for exactly 32
bytes generated by a cryptographically secure random-number generator. The SDK
generates it once per logical assignment operation and reuses the exact
serialized LST body across all DNS
and address attempts, fresh-packet transport retries, temporary-unavailable
retries, and assignment-rate-limit waits inside that operation. A later
top-level operation generates a fresh nonce.

The nonce is authenticated and encrypted as part of the NHP application body.
It is not a credential, is never echoed in LRT, and is never sent to Connector
Authority. The Hub maps the strictly accepted LST mode to one of the three closed
private operations and derives `hub_request_id` locally.

## Derivation

All lengths are unsigned 16-bit big-endian byte lengths. Tags are one byte.

```text
domain = UTF8("layerv:qurl:connector-hub-request-id:v1") || 0x00

preimage =
  domain ||
  0x01 || u16be(len(environment)) || environment ||
  0x02 || u16be(len(operation))   || operation ||
  0x03 || u16be(32)               || authenticated_peer_public_key ||
  0x04 || u16be(32)               || request_nonce

hub_request_id = lowercase_hex(SHA256(preimage))
```

`environment` is exact ASCII matching
`^[a-z](?:[a-z0-9-]{0,30}[a-z0-9])?$`; consumers must not trim, case-fold, or
normalize it. `operation` is the Hub-selected exact ASCII string
`IssueAssignment`, `RefreshAssignment`, or `IssueCredentialRecovery`, never a
free-form caller selector.
The peer key is the raw 32-byte authenticated NHP initiator static public key,
not its Base64 text. `request_nonce` is the raw 32 decoded bytes, not its
base64url text.

The NHP send timestamp, transaction counter, source address, and exact body
digest are deliberately excluded. Fresh outer retries must use new packet
randomness and timestamps while retaining one logical request ID. The
Authority stores an operation-specific semantic request fingerprint alongside
each successful replay record; reusing one nonce with changed request semantics
therefore fails closed instead of becoming a new replay ID.

The Hub alone sends the resulting canonical lowercase 64-hex `hub_request_id`
to the separately permissioned `IssueAssignment`, `RefreshAssignment`, or
`IssueCredentialRecovery`
Authority operation. The SDK, qURL Connector, relay, assigned-cell workers, and
all public NHP bodies neither receive nor derive it.

The `hub_request_id` fixture in `connector_authority_lambda_v1_vectors.json` is
intentionally not this artifact's derived baseline ID. The Authority artifact
tests that the private field remains an opaque canonical lowercase SHA-256 hex
value at its own trust boundary; this artifact alone freezes its derivation.

## Cases

The baseline uses the authenticated agent key and refresh nonce from
`agent_assignment_golden.json`. Five substitution cases change exactly one of
environment, assignment operation, recovery operation, peer key, or nonce.
Consumers independently rebuild every preimage and ID, require every
substitution to differ from the baseline, and require all six IDs to be unique.
These are substitution controls, not a claim to prove SHA-256 collision
resistance. The recovery operation is composed by
`agent_credential_recovery_v1_vectors.json`.
